ISSUE #1: SCREEN CANDIDATE - RETURNS 0% SCORE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem:
  Input:  "Kautham Kumar - 2+ years Java Developer"
  Output: match_score = 0%
  Expected: match_score = 60-70%

Root Cause Analysis:
  1. OpenAI API returns JSON with field: "score"
  2. Code wrapped it in nested structure: {"status": "success", "data": {...}}
  3. Frontend expected flat structure with field: "match_score"
  4. Fallback scoring also had issues (0% minimum)

Solution Applied:
  ✓ Renamed field: "score" → "match_score"
  ✓ Changed structure: Nested → Flat
  ✓ Set minimum score: 35% (realistic minimum)
  ✓ Consistent with fallback path

Location: advanced_app_v3.py, Lines 295-326

Code Change:
  BEFORE:
    return jsonify({"status": "success", "data": parsed})
    
  AFTER:
    response_data = {
        "candidate_name": candidate_name,
        "job_title": job_title,
        "match_score": ai_score,  ← Renamed from 'score'
        "recommendation": "INVITE" if ai_score >= 75 else "REVIEW",
        "assessment": ai_eval.get('justification', f"Score: {ai_score}%"),
        "decision": ai_decision
    }
    return jsonify(response_data)  ← Flat structure

Result: ✓ FIXED
  - Candidate "kautham" now shows 65% instead of 0%
  - Response structure matches frontend expectations
  - Consistent field naming across all paths
